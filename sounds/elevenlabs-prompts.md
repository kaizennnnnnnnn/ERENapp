# Eren — ElevenLabs Sound Prompts

ElevenLabs Sound Effects takes one text prompt → one sound. Generate
each entry below separately.

## Style cue (paste into every prompt for consistency)

> 8-bit / 16-bit chiptune, retro Game Boy / Tamagotchi era, soft and
> kawaii, mono, clean attack, no reverb, normalized volume.

Adjust **duration** in the ElevenLabs duration slider (most UI sounds
are 0.2–0.5 s; fanfares 1–2 s; ambients 20–30 s with the *Loop* toggle
on).

Tip: when ElevenLabs asks for a Prompt Influence, push it high (≥ 70%)
for short UI blips so the model doesn't drift into music.

## Prompting tips for ElevenLabs

1. **Always end the prompt with the style cue** ("8-bit chiptune … no reverb …"). The model otherwise drifts toward orchestral or speech-y output.
2. For **looping ambients**, write `seamless loop` *and* set the duration to a multiple of 4 s — ElevenLabs splices loops better at round numbers.
3. **Generate 3–4 takes per prompt.** Sound generation is stochastic; the second or third take is usually noticeably better.
4. For **UI clicks**, the duration field at 0.1–0.2 s + Prompt Influence ≥ 70 % gives the cleanest blips.
5. **Layer in code, not in prompt.** Don't try to get one prompt to do "purr + chime + bounce" all together — generate two separate sounds and play them simultaneously in the app. Cleaner mix and easier to tune volume.
6. After generating, **normalize to −12 LUFS** (quick pass with `ffmpeg -i in.mp3 -af loudnorm=I=-12:TP=-1.5 out.mp3`) so nothing in your app is jarringly loud or quiet relative to the rest.
